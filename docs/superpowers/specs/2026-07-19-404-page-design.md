# Сторінка 404 — дизайн

## Мета

Надати користувачу зрозумілий український екран помилки замість поточного
рендеру Медіа-центру на неіснуючих URL.

## Рішення

- Створити класичний шаблон теми `wordpress/wp-content/themes/logika-theme/404.php`.
- Зберегти спільні `get_header()` і `get_footer()`, щоб 404 мала той самий header/footer,
  меню, глобальні ACF-значення й доступні контакти, що й інші сторінки.
- Додати окремий `assets/css/404.css`, підключений лише для `is_404()` з `functions.php`.
- Секція 404: темно-фіолетовий фон бренду, біла картка/типографіка, жовтий акцент,
  великий номер `404`, короткий український текст, кнопки на головну та курси.
- Використати `home_url()` і `get_permalink()` для URL; не додавати JS, ACF-поля,
  нові залежності чи нові зображення.

## Межі

- HTTP-відповідь має залишатися `404 Not Found`.
- Увесь текст сторінки — українською.
- Сторінка має бути доступною: один `h1`, зрозумілі назви посилань,
  видимий focus-state, достатній контраст і мобільна версія.
- Поточні незакомічені зміни в worktree не входять до цієї роботи.

## Перевірка

- `php -l` для `404.php` і зміненого `functions.php`.
- `tests/404-page.php` перевіряє наявність шаблону, українських ключових текстів,
  посилань і підключення CSS через статичний контракт.
- `curl -I` та HTML-перевірка реального DDEV URL підтверджують статус 404,
  клас `error404`, контент 404 і stylesheet.
