# Camp Hero Section Design

## Goal

Create `source/camp.html` with the camp hero section from Figma node `449:106441`, reusing the existing Logika landing-page hero implementation and changing only the camp-specific content, image assets, and narrowly scoped layout rules.

## Source of truth

- Local Figma export: `Logika School UX_UI Design (2) (Copy).fig`
- Figma node: `449:106441`, named `Hero section`
- Extracted frame size: `1920x800`
- Existing implementation baseline: `source/index.html` and `source/scss/blocks/sections/banner-section.scss`

## Design

`camp.html` will reuse the existing page partials, `banner-section` markup, `main-form`, shared buttons, Montserrat fonts, global color variables, and JavaScript entrypoints. The section will receive a `camp-hero` modifier rather than changing the global homepage hero.

The left side will use the Figma camp copy:

- `Літній табір на Закарпатті – “Фестиваль професій”`
- `27.06 - 06.07 (перша зміна), 21.07 - 30.07 (друга зміна)`
- `Навчаємо дітей створювати ігри, сайти, застосунки та власні digital-проєкти через практику й сучасні технології.`

The existing hero character slot will become a four-photo circular collage using the four raster assets referenced by the Figma node. The existing Logika character asset remains reused for the form side. The registration form will keep the existing submission structure and behavior, with the Figma camp-specific title and city field represented in the same project-native form styling.

The bottom information bar will reuse the existing `banner-section__bar` pattern and shared icon assets where they exist. Any camp-only CSS will be scoped under `.camp-hero` so homepage/about rendering remains unchanged.

## Asset handling

Copy only the four Figma-referenced images into `source/img/camp/` with semantic filenames. Build output will be generated by the existing Gulp pipeline; generated `build/` files are included only if the repository's current workflow tracks them as build artifacts.

## Responsive behavior

Follow the existing `banner-section` breakpoints and mobile stacking rules. The collage must remain decorative and not reduce form or text readability; on narrow screens it may collapse into a compact row/grid while preserving circular crops.

## Verification

- Run the project's build command and confirm `build/camp.html` and compiled CSS are generated.
- Validate that `source/camp.html` references only existing files.
- Run the project's HTML validator against the generated build output.
- Start the local static preview and visually inspect the camp hero at desktop and mobile widths.
- Confirm the existing homepage hero files are unchanged except for intentionally shared build regeneration.
