# Швидке додавання міста та адреси філії

Дата: 2026-07-17

## Мета

Контент-менеджер має передбачувано додати нове місто на публічну карту, а потім одним переходом створити для нього адресу філії. `city` залишається власником міського контексту, а `branch` — єдиним власником адреси.

## Сценарій редактора

1. Менеджер створює місто, вводить назву та обирає регіон.
2. Нове місто за замовчуванням отримує `city_show_on_map=1`; значення можна явно вимкнути полем «Показувати на карті».
3. Після першого збереження біля поля з’являється кнопка «Додати адресу філії».
4. Кнопка відкриває форму нового `branch` з уже вибраним містом.
5. Для філії менеджер вводить зрозумілу назву та адресу. Філія активна за замовчуванням; координати, телефон і Google Maps URL необов’язкові.
6. Технічні `branch_external_id` і `branch_address_hash` не заважають основному сценарію. Hash формується автоматично зі стабільного city ID та нормалізованої адреси.

## Дані та сумісність

- `city_show_on_map` стає ACF True/False у наявній city group; API і frontend продовжують читати те саме meta-ім’я.
- `branch_city_id`, `branch_address`, `branch_is_active` та інші наявні field keys/meta names зберігаються.
- Форма філії приймає `city_id` лише з валідного запису `city`; довільний post ID ігнорується.
- Імпортовані міста з явним `city_show_on_map=0` не змінюються. Default `1` застосовується тільки до нових записів без збереженого значення.
- Публічна верстка, REST URL і порядок секцій не змінюються.

## Дублікати та ідемпотентність

- Перед публікацією ACF перевіряє канонічний `CitySlug`; друге місто з таким самим URL не публікується та показує посилання на наявний запис.
- Поточний Берестин №877 є канонічним. Непорожні значення з №3003 копіюються лише у порожні поля №877, зв’язки філій переносяться, після чого №3003 переводиться в кошик.
- Міграція має стабільний marker у WordPress options. Повторний запуск не змінює дані та повідомляє нуль операцій.
- Міграція не перезаписує непорожні редакторські значення канонічного міста, крім підтвердженого ввімкнення `city_show_on_map=1` для Берестина.

## Перевірки

- Нове місто з назвою та регіоном повертається city API з `show_on_map=true`.
- Явно вимкнене місто не потрапляє до списку карти.
- Кнопка створення філії передає правильний city ID, а форма показує вже вибране місто.
- Збереження адреси автоматично створює однаковий hash при повторному збереженні.
- Спроба створити друге місто з тим самим канонічним URL блокується.
- Міграція Берестина зберігає непорожні дані, переносить зв’язки та на другому запуску нічого не змінює.
- City API, branch API, city route, ACF Local JSON dry-run і публічний smoke залишаються зеленими.
